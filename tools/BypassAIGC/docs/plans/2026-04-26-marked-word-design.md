# Word 正文提取标记版设计

## 目标

在论文润色页上传 Word 后，系统除了导出正文提取结果 `TXT`，还要生成一个新的标记版 `DOCX`，把所有“被正文提取规则识别到的段落”在原文档中整段浅黄色高亮，方便用户人工核对漏提取或误提取位置。

## 用户可见行为

- 用户上传 `.docx` 或 `.doc`。
- 点击“提取正文”后，系统仍然按现有规则提取正文段落并填入输入框。
- 提取完成后，页面额外提供“下载标记 Word”按钮。
- 下载得到的新文件名为 `原文件名_正文提取标记版.docx`。
- 原始 Word 文件不被修改。

## 识别与标记规则

- 目录仍然跳过。
- 从真实正文中的第一个 `1.1` 开始识别。
- 仅提取正文段落，不把章节标题写入 `TXT`。
- 图题、表题、页码、目录项、总结/结论/参考文献/致谢/附录等停止或过滤规则沿用当前实现。
- 标记版 Word 中，仅对“被提取到的正文段落”做浅黄色高亮；未提取内容保持原样，便于肉眼比对。

## 技术方案

### 后端

- 继续复用现有 `docx_body_extractor.py` 的正文识别逻辑。
- 提取时保留原始 `paragraphs` 索引，用它们反向定位 Word 中对应段落。
- 新增一个“生成标记版 Word”的导出函数：
  - 读取原始 `docx`
  - 根据已识别的段落索引定位原段落
  - 对段落内所有文本 run 施加浅黄色底纹
  - 输出新的 `docx` 字节流
- 新增下载接口，直接返回 `application/vnd.openxmlformats-officedocument.wordprocessingml.document`。

### 前端

- 论文润色页保留现有上传和 `TXT` 下载入口。
- 提取成功后显示“下载标记 Word”按钮。
- 点击按钮时，把当前已上传的同一份 Word 文件再次发给后端，下载返回的标记版 `docx`。
- 上传新文件时清空上一份提取结果，避免结果与下载文件错配。

## 错误处理

- 上传空文件、超出大小限制、文件格式不支持时，沿用现有错误提示。
- `.doc` 转换失败时返回清晰错误。
- 若文档中未识别到真实正文段落，不生成标记版 Word。

## 验证

- 用带目录、正文、标题、图题、参考文献的样例文档验证：
  - `TXT` 只包含正文段落
  - 标记版 Word 仅高亮正文段落
  - 标题、目录、图题、表题不高亮
- 重新打包 Windows `exe` 并做启动验证。
