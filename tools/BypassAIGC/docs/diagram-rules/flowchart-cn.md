# 中文论文流程图绘制规范

本规范用于毕业论文中的中文软件流程图、程序流程图和功能流程图绘制，默认采用黑白线稿风格，适合直接插入论文正文。

## 总体风格

- 白色背景，黑色边框与黑色连线。
- 不使用彩色、渐变、阴影、装饰性图标。
- 默认不在图内额外放置大标题，图名由论文正文中的图题承担。
- 整体采用自上而下布局，层级清晰，留白均匀。

## 节点规范

- `开始` 与 `结束` 必须使用终止框。
- 普通处理步骤使用矩形框。
- 条件判断使用菱形框。
- 同一类型节点的框大小应保持一致。
- 文本使用中文居中排布，字体以宋体、微软雅黑或黑体风格为准。

## 连线规范

- 仅使用直角连接线，不使用斜线或曲线。
- 连线不能交叉。
- 连线不能穿过节点。
- 连线不应与无关节点边框重合。
- 判断分支应就近回接到有意义的流程节点，不引入无必要的占位节点。
- `是`、`否` 等分支文字应标注在对应分支附近。

## 布局规范

- 主流程沿中轴线自上而下展开。
- 回路优先布置在节点外侧，避免压到主流程节点。
- 节点之间的纵向间距保持一致，避免上下拥挤。
- 分支节点与回路节点之间保留足够水平距离，确保回路线可独立布线。

## 示例要求

- 默认参考示例图：`flowchart-cn-example.png`
- 若用户提供新的参考图片，则优先遵循用户图片。
- 若用户明确要求图内标题、彩色主题或特殊节点样式，以用户要求为准。
