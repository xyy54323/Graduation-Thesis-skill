# 检测报告处理与降 AIGC 改写

当用户提供 PaperPass、SpeedAI 等 AIGC 检测报告，或要求“降 AI”“降 AIGC”“根据检测报告改写”时，读取本文件。

本模块优先使用本 skill 自带工具 `tools/BypassAIGC/AI学术写作助手.exe` 完成报告解析、片段提取、降 AIGC 改写和 Word 导出。AI 的职责不是手工逐段凭感觉改写，而是启动工具、准备文件、确认提取范围、执行优化、导出结果并记录处理说明。

完成任一流程后，必须按 [降AIGC验收清单.md](降AIGC验收清单.md) 复核；其中正文参考文献引用跳转恢复是导出后的立即动作，不能拖到说明文件之后。

## 1. 工具位置与运行前检查

工具目录：

```text
tools/BypassAIGC/
├── AI学术写作助手.exe
├── .env
└── ai_polish.db
```

运行前检查：

- 确认 `AI学术写作助手.exe`、`.env`、`ai_polish.db` 在同一目录。
- 确认 `.env` 已由用户配置好真实可用的模型名称、Base URL 和 API Key；AI 不要替用户编造或猜测密钥。
- 目标论文必须是 `.docx`，工具当前只支持 docx 文档。
- 检测报告必须是 `.pdf`，当前重点支持 PaperPass 和 SpeedAI。
- 需要导出或写回 Word 时，本机应安装 Microsoft Word。

启动方式：

```powershell
Start-Process -FilePath "tools/BypassAIGC/AI学术写作助手.exe"
```

启动后默认访问：

```text
http://localhost:9800
```

如果 9800 端口不可用，先查看工具日志或 `.env` 中的 `SERVER_PORT`，再访问对应端口。

## 2. AI 使用工具的基本原则

- 优先通过工具页面完成降 AIGC，不要绕过工具自行整篇改写。
- 检测报告只用于定位疑似片段和处理优先级，论文事实仍以项目源码、论文原文和用户确认材料为准。
- 不承诺一定降到某个具体分数。
- 不为了降分编造项目没有实现的功能。
- 不处理摘要，除非用户明确要求；中英文摘要容易出现语义不一致问题。
- 不主动修改目录、声明、封面、参考文献、图题、表题和学校固定格式文本。
- 改写后仍保持本科毕业论文口吻，不能过度口语化。

## 3. 功能选择

工具首页提供两个入口：

| 入口 | 使用场景 | 输入文件 | 输出结果 |
| --- | --- | --- | --- |
| 整篇提取优化 | 用户没有检测报告，或要求先对正文整体降 AI | 原始论文 `.docx` | 提取正文、标记版 Word、优化任务、导出 Word |
| AIGC片段优化 | 用户提供 PaperPass 或 SpeedAI 检测报告 | 原始论文 `.docx` + 检测报告 `.pdf` | 疑似片段、片段标记版 Word、优化任务、导出 Word |

选择规则必须明确：

- 只要用户提供了 PaperPass 或 SpeedAI 的 PDF 检测报告，就使用“AIGC片段优化”。
- 没有 PaperPass 或 SpeedAI 的 PDF 检测报告时，使用“整篇提取优化”。
- 不要在已有 PDF 检测报告的情况下默认整篇优化，除非用户明确要求。

## 4. AIGC片段优化流程

适用于用户提供检测报告的情况。

1. 打开 `http://localhost:9800`。
2. 选择功能卡片“AIGC片段优化”。
3. 上传原始论文 `.docx`。
4. 上传对应的 PaperPass 或 SpeedAI `.pdf` 检测报告。
5. 点击提取按钮，让工具解析报告并匹配 Word 中的疑似片段。
6. 查看报告指标和匹配结果：
   - PaperPass 通常关注总体疑似率、加权疑似率、片段风险等级。
   - SpeedAI 通常关注总体、高度、中度、轻度等状态。
7. 下载“标记 DOCX”，用浅黄色标记确认本次实际会被优化并写回的位置。
8. 如果标记位置明显错误或数量异常，停止处理并向用户说明；不要强行优化。
9. 标记确认无误后，点击“开始优化”。
10. 等待任务完成，进入“历史记录”或当前记录详情页查看优化结果。
11. 在详情页导出 `.docx`，工具会把优化后的片段写回原 Word 版式。
12. 导出后立即恢复正文参考文献引用跳转；如果引用编号变成普通文本，先按文末参考文献编号恢复交叉引用或内部超链接，再写说明文件。

导出前确认：

- 会话状态为“已完成”。
- 会话来自 Word 提取，详情页提示可导出 `.docx`。
- 导出时确认学术诚信提示。
- 导出后必须立即处理正文参考文献引用跳转；如果引用编号变成普通文本，必须按 `references_word/Word回写与交付检查.md` 恢复交叉引用或超链接，不能只记录问题。

## 5. 整篇提取优化流程

适用于没有检测报告、用户要求先整体降 AI 的情况。

1. 打开 `http://localhost:9800`。
2. 选择功能卡片“整篇提取优化”。
3. 上传原始论文 `.docx`。
4. 点击正文提取，让工具自动跳过摘要、目录、关键词、标题、图题和表题，提取正文段落。
5. 下载“标记 DOCX”，确认浅黄色段落是否为实际要优化的正文。
6. 如果摘要、目录、图题或表题被误提取，先停止并修正工具或让用户确认，不要直接优化。
7. 标记确认无误后，点击“开始优化”。
8. 任务完成后进入详情页，导出 `.docx`。
9. 导出后立即恢复正文参考文献引用跳转，确认正文 `[1]`、`[2]` 等引用编号仍能跳到文末对应参考文献。

整篇优化会改动范围更大，只有用户明确同意整体处理时使用。对已经人工改好的章节，优先使用片段优化或让用户确认处理范围。

## 6. 页面操作与浏览器自动化

需要 AI 自动操作页面时，优先使用浏览器自动化工具打开本地地址：

```text
http://localhost:9800
```

关键页面与按钮：

- 首页默认是降 AIGC 页面。
- “整篇提取优化”：进入整篇 docx 提取流程。
- “AIGC片段优化”：进入 docx + pdf 报告联合提取流程。
- “下载标记 DOCX”：下载本次提取范围标记版，用于核对。
- “开始优化”：创建优化任务。
- “历史记录”：查看已有任务。
- 历史记录中的会话卡片：进入详情页。
- 详情页“导出”：导出 txt 或 docx，Word 来源任务优先导出 docx。

若页面操作失败，可改用接口辅助排查，但正式流程仍以页面结果和导出文件为准。

常用接口：

| 接口 | 用途 |
| --- | --- |
| `POST /api/optimization/extract-docx-body` | 上传 docx 并提取正文 |
| `POST /api/optimization/mark-docx-body` | 上传 docx 并下载正文标记版 |
| `POST /api/optimization/extract-aigc-report` | 只解析 AIGC 检测报告 |
| `POST /api/optimization/extract-aigc-segments` | 上传 docx + pdf，提取并匹配疑似片段 |
| `POST /api/optimization/mark-extracted-word` | 根据提取 token 下载标记版 Word |
| `POST /api/optimization/start` | 启动优化任务 |
| `GET /api/optimization/sessions` | 查看历史任务 |
| `GET /api/optimization/sessions/{session_id}` | 查看任务详情 |
| `POST /api/optimization/sessions/{session_id}/export` | 导出优化结果 |

启动优化时的关键字段：

```json
{
  "original_text": "工具提取出的正文或疑似片段文本",
  "processing_mode": "paper_polish_enhance",
  "extraction_token": "工具提取阶段返回的 token"
}
```

保留 `extraction_token` 很重要；没有它，工具无法把优化结果准确写回原 Word 段落。

## 7. 处理结果记录

每次使用工具降 AIGC 后，生成或更新以下说明文件：

- `降AIGC分析报告.md`
- `降AIGC修改说明.md`

分析报告记录：

- 检测平台：PaperPass、SpeedAI 或其他。
- 检测报告文件名。
- 原始论文文件名。
- 总体指标：总体疑似率、加权疑似率，或 SpeedAI 的总体/高度/中度/轻度状态。
- 工具匹配到的段落数量。
- 未匹配片段数量。
- 是否下载并核对过标记版 Word。

修改说明记录：

- 使用的工具路径。
- 使用的功能入口：整篇提取优化或 AIGC片段优化。
- 优化模式：润色、降 AI、或先润色再降 AI。
- 导出的文件路径。
- 导出后已立即恢复正文参考文献引用跳转，并记录恢复结果。
- 未处理内容及原因。
- 需要用户重新检测的提醒。

## 8. 复检流程

第一轮：

1. 使用工具处理检测报告。
2. 下载标记版 Word，确认处理范围。
3. 开始优化并导出 Word。
4. 立即恢复正文参考文献引用跳转，并按 [降AIGC验收清单.md](降AIGC验收清单.md) 复核。
5. 记录分析报告和修改说明。
6. 让用户重新提交检测。

第二轮及以后：

1. 使用最新检测报告。
2. 优先使用“AIGC片段优化”。
3. 只处理最新报告仍然标出的片段。
4. 导出后立即恢复正文参考文献引用跳转，并按 [降AIGC验收清单.md](降AIGC验收清单.md) 复核。
5. 不重复整篇改写已经通过或用户确认的内容。

如果多轮报告冲突，以最新报告为准，同时保留旧报告中的修改记录，方便追踪。

## 9. 异常处理

- 工具打不开：检查 exe、`.env`、数据库是否同目录，检查端口和日志。
- 页面显示 API Key 未配置：让用户填写 `.env` 或管理配置；不要替用户编造密钥。
- 报告解析失败：确认是否为 PaperPass 或 SpeedAI PDF；若不是，先说明当前工具可能不支持该报告格式。
- 匹配片段过少：下载标记版 Word 核对，必要时让用户提供报告截图或改用整篇提取优化。
- 优化结果白话味太重：优先调整工具内提示词，再重新运行任务。
- 导出 Word 失败：确认会话是否来自 Word 提取、Microsoft Word 是否安装、目标文件是否被占用。
- 参考文献引用不能跳转：降 AIGC 导出可能导致正文引用域丢失，必须立即按原文献编号逐项恢复交叉引用或内部超链接，修复完成后再写说明文件或交付。
- 输出仍出现禁用连接词：先确认运行的是 `tools/BypassAIGC/AI学术写作助手.exe`，不是旧的 `package/dist` 构建版。

## 10. 安全与版本控制

- 运行工具时，`.env` 需要由用户自己配置真实可用的 API Key、Base URL 和模型名称。
- 提交或发布 skill 仓库时，不能上传用户真实密钥；仓库中的 `.env` 应恢复为模板或占位配置。
- 仓库只保留工具运行必需文件：exe、`.env` 配置模板、初始数据库。
- 本地测试报告、论文样例、日志、缓存、源码构建目录不要上传。
- 若需要备份旧数据库或用户文档，放入工作区外或专门的备份文件夹，并按“工具名/文件类型/时间”分类。
