# 降 AIGC 验收清单

每次使用 `tools/BypassAIGC/AI学术写作助手.exe` 完成降 AIGC 后，必须先按本清单复核，再写说明文件或交付给用户。

## 1. 输入与模式

- 目标论文是 `.docx` 文件。
- 用户提供 PaperPass 或 SpeedAI 的 PDF 检测报告时，使用“AIGC片段优化”。
- 没有 PaperPass 或 SpeedAI 的 PDF 检测报告时，使用“整篇提取优化”。
- 已有可支持的 PDF 检测报告时，不默认整篇优化，除非用户明确要求。
- 摘要、目录、图题、表题、参考文献和学校固定格式文本默认不处理。
- Word 回写、格式检查或最终交付前，已主动进入 AIGC 优化检查流程；若未执行，必须记录工具、配置、权限或用户要求等阻塞原因。

## 2. 工具执行

- 使用 skill 仓库中的 `tools/BypassAIGC/AI学术写作助手.exe`。
- `.env` 由用户自己配置真实可用的 API Key、Base URL 和模型名称，AI 不编造密钥。
- 优化前已下载并核对标记版 Word，确认浅黄色标记范围正确。
- 优化任务状态为“已完成”后，再从详情页导出 `.docx`。
- 导出的 `.docx` 来自带有 `extraction_token` 的 Word 来源会话。

## 3. 导出后必须立即修复

- 导出 Word 后，立即检查正文参考文献引用是否还能跳转。
- 如果正文 `[1]`、`[2]` 等引用编号变成普通文本，必须先恢复交叉引用域或内部超链接。
- 修复时按文末参考文献编号逐项对应，不改变正文含义和参考文献列表内容。
- 修复完成后，逐项验证正文引用编号能跳到文末对应参考文献，不只抽查。
- 不允许只把引用跳转丢失写进说明文件后直接交付；无法恢复时停止交付并列出阻塞引用编号。

## 4. 结果说明

- 生成或更新 `降AIGC分析报告.md`。
- 生成或更新 `降AIGC修改说明.md`。
- 报告中记录使用入口：整篇提取优化或 AIGC片段优化。
- 报告中记录检测平台、报告文件、原始 Word、匹配片段数、未匹配片段数。
- 报告中记录标记版 Word 是否已核对。
- 报告中记录导出文件路径和参考文献引用跳转逐项修复结果。

## 5. 异常处理

- 工具无法解析检测报告时，先说明当前报告格式可能不支持，不强行改写。
- 标记版 Word 范围明显异常时，停止优化并向用户说明。
- 导出失败时，检查 Word 是否被占用、任务是否来自 Word 提取、会话是否已完成。
- 引用跳转无法恢复时，停止交付并报告具体阻塞位置。
